== Contexto y alcance

=== Contexto de negocio
[mermaid, "business-context", png]
----
flowchart LR
    Jugador["Jugador"] -- Datos inicio sesión --> YOVI(("YOVI"))
    Jugador -- Acciones de juego --> YOVI
    Bot["Bot Externo"] -- Movimientos via API --> YOVI
    
    YOVI -- Acceso permitido/denegado --> Jugador
    YOVI -- Estado del juego --> Jugador
    YOVI -- Resultado partida --> Jugador
    YOVI -- Estado partida via API --> Bot
----

[cols="1,3"]
|===
|Elemento |Descripción

|Jugador
|Usuario humano que interactúa con el sistema YOVI a través de la aplicación web para jugar partidas del juego Y. Se autentica en el sistema y realiza movimientos dentro de partidas iniciadas a través de la aplicación. Después de hacer movimientos recibirá el resultado que eso produce en la partida.

|Bot Externo
|Bot automatizado que utiliza un API para conectarse a YOVI para jugar partidas sin necesidad de interacción humana directa.

| YOVI
|Sistema completo de juegos tipo Y. Engloba la aplicación web en TypeScript y el módulo de validación en Rust. Gestiona autenticación de usuarios, lógica de juego, validación de partidas y sugerencias de movimientos.
|===

=== Contexto técnico

[mermaid, "technical-context", png]
----
flowchart LR
  U[Usuario]
  W[Aplicación Web]
  US[Servicio de Usuarios]
  G[Motor de Juego]
  DB[(Base de datos)]

  U -->|HTTP mediante navegador| W
  W -->|REST / JSON| US
  US -->|persistencia de datos| DB
  W -->|REST / JSON| G
  G -->|estado y resultados| W
----

En este contexto, el usuario interactúa con el sistema mediante una aplicación web.
La aplicación web actúa como punto de entrada al sistema y se comunica con los servicios internos a través de APIs REST.
El servicio de usuarios gestiona la información de registro y autenticación, mientras que el motor de juego se encarga
de la lógica de las partidas. La persistencia de datos se realiza mediante una base de datos.

==== Mapeo de entradas y salidas a canales

[cols="1,2,2", options="header"]
|===
|Entrada / Salida |Canal |Descripción

|Datos de registro
|HTTP (REST / JSON)
|Envío de datos del usuario desde la aplicación web al servicio de usuarios

|Datos de usuario
|Conexión con base de datos
|Almacenamiento y recuperación de información de usuarios

|Acciones de juego
|HTTP (REST / JSON)
|Envío de jugadas desde la aplicación web al motor de juego

|Estado y resultados de la partida
|HTTP (REST / JSON)
|Respuesta del motor de juego a la aplicación web para su visualización al usuario
|===
