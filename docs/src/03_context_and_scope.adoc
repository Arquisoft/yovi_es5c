== Contexto y alcance

=== Contexto de negocio
[mermaid, "business-context", png]
----
flowchart LR
    Jugador["Jugador"] ----> YOVI(("YOVI"))
    
    Bot["Bot Externo"] ----> YOVI
    
    YOVI ----> Jugador

    YOVI ----> Bot
----

[cols="1,3"]
|===
|Elemento |Descripción

|Jugador
|Usuario humano que interactúa con el sistema YOVI a través de la aplicación web para jugar partidas del juego Y. Se autentica en el sistema y realiza movimientos dentro de partidas iniciadas a través de la aplicación. Después de hacer movimientos recibirá el resultado que eso produce en la partida.

|Bot Externo
|Bot automatizado que utiliza un API para conectarse a YOVI para jugar partidas sin necesidad de interacción humana directa.

| YOVI
|Sistema completo de juegos tipo Y. Engloba la aplicación web en TypeScript y el módulo de validación en Rust. Gestiona autenticación de usuarios, lógica de juego, validación de partidas y sugerencias de movimientos.
|===

=== Contexto técnico

----

  U -->|HTTP/HTTPS | W

  W -->|REST / JSON| API

  W -->|REST / JSON | G
  G -->|REST / JSON | W

  W -->|REST / JSON| DB
----

[cols="1,3", options="header"]
|===
|Nodo |Descripción

|Usuario
|Persona que accede a Yovi mediante un navegador web para registrarse, iniciar sesión y jugar partidas del juego Y.

|Aplicación Web / Yovi
|Aplicación accesible vía web que ofrece la interfaz de usuario, coordina la lógica del sistema y actúa como punto de entrada a los servicios internos y al API para bots.

|API externa para bots
|Interfaz pública expuesta por la aplicación que permite a clientes externos interactuar con el sistema para consultar información y jugar partidas de forma automatizada.

|Bot / cliente externo
|Cliente automatizado externo que consume el API para interactuar con el sistema, incluyendo la ejecución de partidas contra la aplicación.

|Servicio de juego / Rust
|Módulo encargado de validar el estado de las partidas y de sugerir el siguiente movimiento según las reglas del juego Y.

|Base de datos / MongoDB
|Sistema de persistencia que almacena la información de usuarios, partidas, estadísticas e histórico de participación.
|===

==== Mapeo de entradas y salidas a canales

[cols="1,2,2", options="header"]
|===
|Entrada / Salida |Canal |Descripción

|Registro/login
|Usuario → Aplicación Web (HTTP/HTTPS)
|El usuario crea cuenta e inicia sesión desde el navegador.

|Consulta de histórico/estadísticas
|Usuario → Aplicación Web (HTTP/HTTPS)
|El usuario consulta partidas jugadas, ganadas/perdidas, etc.

|Jugada del usuario
|Usuario → Aplicación Web (HTTP/HTTPS)
|El usuario realiza un movimiento desde la UI.

|Play (bot)
|Bot → API externa (HTTP/HTTPS)
|El bot envía `position` en notación YEN y recibe el siguiente movimiento.

|Siguiente movimiento / validación
|Aplicación Web ↔ Servicio Rust (REST + JSON/YEN)
|La app delega en Rust el cálculo/validación y recibe respuesta.

|Persistencia
|Aplicación Web → Base de datos
|Guardar/leer usuarios, partidas y estadísticas.
|===
