Tabla 1. Bloques de construcción contenidos (Nivel 1)

[cols="1,3", options="header"]
|===
|Bloque |Descripción

|Stakeholder: Usuario
|Persona que accede a Yovi desde un navegador para registrarse y jugar partidas.

|Stakeholder: Bot
|Cliente externo que interactúa con el sistema mediante el API expuesto para bots.

|Aplicación Web
|Punto de entrada del sistema. Ofrece la UI y expone el API para bots. Orquesta llamadas al servicio de usuarios y al servicio de juego.

|Servicio de Usuarios
|Gestiona registro/autenticación y almacenamiento de información de usuarios y estadísticas/histórico.

|Servicio de Juego (Rust)
|Implementa la lógica del juego Y: Plataforma de juego automatizada y cálculo del siguiente movimiento según estrategias.

|Base de datos
|Persistencia de usuarios, resultados de partidas e información estadística/histórico.
|===

Tabla 2. Bloques de construcción contenidos (Aplicación Web)

[cols="1,3", options="header"]
|===
|Bloque |Descripción

|UI / Vistas
|Pantallas y componentes que permiten al usuario registrarse, iniciar partidas y jugar.

|Lógica que valida peticiones, aplica reglas de aplicación y coordina llamadas a servicios.

Tabla 3. Bloques de construcción contenidos (Servicio de Juego)

[cols="1,3", options="header"]
|===
|Bloque |Descripción

|Interfaz Web/CLI Usuario
|Expone operaciones del servicio para comunicarse con el Usuario de la aplicación (actualmente por línea de comandos, API Web en el futuro).

|Motor de juego
|Modelo de estado, reglas, validación y aplicación de acciones.

|API de servidor de bots
|Punto de entrada para bots remotos; expone estado, recibe y valida elecciones.

|Gestor de bots
|Registro, creación y selección de instancias de bots y políticas de elección.

|Bots
|Implementaciones de estrategias (aleatorio, heurístico, etc.) que proponen acciones.

|Notación y I/O
|Serialización/deserialización y representación textual del estado/jugadas.

|Manejo de errores
|Tipos y propagación de errores compartidos por los componentes.
|===


=== Nivel 3

==== Whitebox: Core del juego

[mermaid, "bb-level3-core", png]
----
flowchart TB
  subgraph Engine [Motor de juego]
    Coord[Topologia / Coordenadas]
    State[Estado de la partida]
    Validator[Validador / Movimiento]
    Victory[Detector de victoria]
    Players[Gestión de jugadores]
    Render[Representación / Opciones]
  end

  Coord -->|define vecindad y conversiones| State
  Coord -->|vecindad| Validator
  State -->|consulta| Validator
  State -->|analiza conexiones| Victory
  Validator -->|aplica mutaciones| State
  Players -->|consulta/actualiza| State
  State -->|serialización/representación| Render
----

Tabla 4. Bloques de construcción contenidos (Core del juego)

[cols="1,3", options="header"]
[cols="1,3", options="header"]
|===
|Bloque |Descripción

|Topología / Coordenadas
|Funciones y utilidades para coordenadas y vecinos del tablero (src/core/coord.rs). No guarda estado.

|Estado de la partida
|Estructura central que mantiene posiciones, turno y dimensiones; contiene la lógica de consulta del estado (src/core/game.rs).

|Validador / Movimiento
|Validación de intenciones de movimiento y aplicación de mutaciones sobre el estado (src/core/movement.rs, src/core/action.rs).

|Detector de victoria
|Algoritmos que analizan conexiones en el tablero para decidir condiciones de victoria (implementado en funciones vinculadas a src/core/game.rs).

|Gestión de jugadores
|Modela jugadores y conjuntos de jugadores, APIs para añadir/consultar participantes (src/core/player.rs, src/core/player_set.rs).

|Representación / Opciones
|Opciones de render y serialización del estado para salida/IO (src/core/render_options.rs).
|===
